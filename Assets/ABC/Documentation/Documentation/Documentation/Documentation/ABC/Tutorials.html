Creating Your First Ability

Overview

This tutorial will take you through the very basics of setting up your first ability. The steps below are basic and use minimal functionality. By the end of the tutorial you will have 2 gameobjects on the scene with one object activating an ability which will hit the other, adding explosive force. A more in depth guide on setting up a scene with animations, GUI, targetting and abilities with effects can be found in the Quick Start tutorial. More advanced guides can be found in the tutorial videos which will be coming soon.

Adding The GameObjects & Components

First we will be adding the player object and setting up the camera and movement. Follow the steps below to accomplish this.

1. Load up a new scene and add a Terrain by going to GameObject > 3D Object > Terrain

2. Add 2 Capsule objects to the scene placing them opposite each other, name one "Player" and the other "Enemy". Next Add a Rigibody Component to the Enemy.

3. Add the ABC_Controller component to the Player Object and the ABC_StateManager component to the enemy. The ABC_Controller will allow us to setup and manage the ability. The ABC_StateManager is required to interact with the ability.

Creating The Ability

The next step is to create the ability

4. With the Player gameobject selected in the hierarchy go back to the ABC_Controller and select the "Ability Manager" button to open the Ability Editor screen. From here Click the "Add New Ability" button. This will bring up new screens ready to customise the ability.

5. In the config that has just loaded, add the name "First Ability", choose an Icon Image like "Default Projectile Icon", Write in the text area for a description and press the circle next to the "Main Graphic" setting and search and select "ABC_DefaultProjectile". A screenshot below will display how the settings should look.

6. Next it's time to setup the key which will activate the ability. Press the "Key & Combo" button in the menu to the left. This will bring up a new config screen. In this section press the 'Key' setting dropdown and select a keyboard button, for this tutorial we have gone with "F".

7. Once the above has been completed press the "Position & Travel" tab at the top. This section defines where the ability starts from and how it will travel. For this tutorial we will make the ability travel to the mouse location however there is plenty more options you can play with. Another selection which works for this tutorial is "Forward" which makes the ability travel in a straight line. The selected target options requires the targetting functionality which is covered in the Quick Start tutorial. Go ahead and select "Mouse Forward". The "Starting Position" setting can stay on "Self" which means it will travel from the Player object but like travel types there is plenty more selection to pick from, like making the ability start from a specific object.

8. Now is time to configure the collision and impact, press the "Collision & Impact" tab at the top which will load the "Collider" settings. In this screen make sure "Add Collider" is ticked and "Is Trigger" is not ticked as shown in the screenshot below.

9. Next press the "Collision" button to the menu to the left, in this section we will allow the ability to collide with the Terrain by unticking the "Ignore Terrain" box.

10. Now we will add the explosive force, select the "Impact" button to the menu to the left. Once the screen loads tick "Splash Explosion" then enter 5 in the Radius, 300 in the Explosion Power and 20 in the Uplift. You are welcome to play with these values for more explosive effects.

11. The last step is to add the "Ability End" graphic which will show an explosion when the ability hits the enemy. Press the "Aesthetic & Animation" tab at the top, then the "Ability End" button to the left. On this screen tick the "Use Ability End Aesthetics" then sellect the circle next to "Main Graphic" and find the ProjectileImpact prefab. Once done enter 3 for the Graphic Duration.

In Game

The tutorial has been completed, Run the game and aim the mouse and press the F key or the key you selected to activate the ability. The ability will activate and travel towards the mouse position, on impact an explosive force will hit the enemy. Now this is completed I recommend following the Quick Start Tutorial. Which goes through more functionality including adding ability effects to damage the enemy, animations and GUI to display health/mana.

Quick Start

Overview

This quick start tutorial will take you through the basics of ABC. We will set up a player, an enemy and a Fireball ability. The tutorial also includes Targetting configuration, Ability Effects, Animations and Health/Mana GUI setup. The player will be able to press a trigger to activate the Fireball sending it to the enemy inflicting damage. The tutorial is sectioned over 30 steps.

Adding & Moving The Player

First we will be adding the player object and setting up the camera and movement. Follow the steps below to accomplish this.

1. Load up a new scene and add a Terrain by going to GameObject > 3D Object > Terrain

2. Navigate to Assets\ABC\Demos\BattleChamber\Characters\Player and Drag the 'EthanBase' prefab to the scene. This prefab has the Animator component with the PlayerAniController already set and the movement script 'Input Third Person'. The movement component and Animation Controller are part of the example demo and not supported in ABC but you are free to use them for any project.

3. Next step is to setup the camera. Delete the default Main Camera from the scene and drag in the 'CameraBase' prefab located at: ABC\Demos\BattleChamber\Camera. This camera is used for the BattleChamber demo and is not supported in ABC but you are free to use it for any project.

4. Once the camera is in the scene select it and expand the Camera Base Third Person component. This script is part of the example demo and not supported in ABC but you are free to use it for any project. Once selected drag the 'EthanBase' object from your scene into the Follow Target setting. The camera comes with some default audio music If you don't want this then remove the Audio attached to the camera base object.

5. Press play and you can now move the camera round with the mouse and the player with WSAD.

Setting Up The Players Health

Now that player can move around is time to do basic setup of health.

6. We will now setup health for the player. Select the EthanBase Object and add the ABC_StateManager Component. Once added press the State Manager button to open the StateManager Editor Screen. The ABC_StateManager component is which can configure health, hit animations and more detailed in the documentation found by clicking this link: State Manager Config Documentation.

7. On the new editor screen select the Health button to the left, this will load all the health settings ready to be changed. From here you can configure the current health, max health and how much health regens each second. The defaults can be used for this tutorial but you are welcome to change and play with the values.

8. Next is to configure the health GUI to let the player how much health is remaining on the enemy. Select the 'Add Health GUI' button to make a new section appear below.

9. At this point the settings requires a Unity Slider and Text Object which it will use to display the current health of the player. For this tutorial we are going to use the defaults. To load the defaults press the following icon next to each setting. You will notice that this automatically adds the ABC_GUI prefab to the scene. You are welcome to change, move and play with the Slider and Text objects we just automatically loaded or use your own.

10. Now that the ABC_GUI prefab has been loaded we are going to deactivate some of the objects in the inspector we won't be using in this tutorial. Go ahead and disable the following: healthReductionImage, ScrollEffectLog, AbilityLog, AbilityGroupPoints, Stats, CurrentScrollAbility, CurrentScrollAbilityAmmo

Setting Up The Players ABC Controller

The next step is to setup the ABC Controller. For this basic tutorial we will use this component to setup mana, GUIs, targetting and an ability. This is only a small amount of functionality which the component is capable of and more details on further functionality can be found in the documentation.

11. Select the EthanBase Object and add the ABC_Controller Component. Once added press the "Controller Manager"" button to open the ABC Controller Config Editor Screen. More details on what can be configured in this screen can be found in the documentation by clicking this link: ABC Controller Documentation.

12. On the new editor screen select the Mana button to the left, this will load all the mana settings ready to be changed. From here you can configure the current mana, max max and how much mana regens each second. The defaults can be used for this tutorial but you are welcome to change and play with the values.

13. Next is to configure the Mana GUI to let the player how much Mana is remaining. Select the 'Add Mana GUI' button to make a new section appear below. Just like when setting up health in the previous section, we will be using the defaults. Click the load default icon next to each setting. This will automatically place the Slider and Text object from the ABC_GUI Prefab which was loaded earlier. You are welcome to change this or use your own Slider/Text Objects.

14. Now we will add the preparing GUI ready for when we create the ability. Select the 'General & Error' button to change the setting screen. Scroll down to the bottom and press the load default icon next to Preparing Slider and Preparing Text setting. This will automatically add the correct objects found in the the ABC_GUI prefab which was loaded earlier in the tutorial. You are welcome to change, move and play with the Slider and Text object or use your own.

15. The last step is to setup the auto-targetting. Press the Target Tab at the top of the editor screen. For this tutorial we will make use of the auto targetting rather then select targetting so for this section all we need to add is the indicator. Press the load default icon next to the Target Indicator setting. Next make sure the Indicator Scale is set to 2.5

16. Next press the Auto & Tab Target button to the left of the editor screen and tick the Auto Target Select setting.

Creating A Fire Spell

Now the initial setup has been completed we can create the players first Ability, a fire spell. The ability will be setup to prepare for a few seconds and then travel towards the players target, causing damage on impact.

17. With the EthanBase gameobject selected in the hierarchy go back to the ABC_Controller and select the "Ability Manager" button to open the Ability Manager Editor screen. From here Click the "Add New Ability" button. This will bring up new screens ready to customise the ability.

18. We will now do the general setup for the ability, follow the screenshot below to configure all the settings including the name, icon, graphic, prepare time (including show on gui), duration, manacost and cooldown. The graphic and icons are assets which are included with ABC.

19. The next section to configure is the button trigger to activate the ability. Press the Key & Combo button to the side to change the settings page. From here the only setting to change is the Key, feel free to select which ever key you want to activate the ability. In this example we have gone with 'F'

20. Next is to add the effect to reduce the enemies health. Press the Effect button to the side to change the settings page. Then click the dropdown and select 'Adjust Health' once selected press the + button. a box below will appear where additional configuration for the effect can be done. There is no limit to how many effects can be added to an ability. enter -50 in the potency setting in the effect config box that appeared. This is all that's required for this tutorial but note, expanding this box allows for much more configuration. Detailed in the documentation which can be found by clicking the following link: Ability Effects Overview

21. From here press the Position & Travel tab to load the next set of settings we are going to change. This section determines where the ability will start from and how it will travel i.e to a selected target. Other options exist like send an ability in a straight line or to mouse location. Follow the screenshot below to configure all the settings which will make the ability travel to the selected target, starting from ethans position midway up. Note: There is other options which allows the starting position to be from a set gameobjects position and more.

22. The last section to configure for the ability is the aesthetics and animations. First is the preparing. Select the 'Aesthetics & Animations' tab which will automatically load the preparing section. Tick the 'Use Preparing Aethetics' and further config will appear. Follow the screenshot below entering all the settings. The Animator Parameter links directly with the Animator controller which has already been setup on Ethan. If you have used your own animation controller then enter the name of the animation parameter the type and the input (on/off etc).

23. Next is the initiation animations and aesthetics. Select the Initiation menu button to the left to load the next config then just like above copy the screenshot to configure the ability initiation following the same instructions.

24. The last to configure is the Ability End. Select the 'Ability End' button on the menu to the left and once again follow the screenshot to configure.

Adding & Setting Up The Enemy

The last part of the tutorial is setting up the enemy.

25. Navigate to Assets\ABC\Demos\BattleChamber\Characters\Enemies\ShadowWolf\Model and Drag the 'ShadowWolfBase' prefab to the scene. This prefab has the Animator component with the MonsterAniController already set and a Capsule Collider component. Animation Controller are part of the example demo and not supported in ABC but you are free to use them for any project.

26. We will now setup hit animations and health for the enemy. Select the ShadowWolfBase and add the ABC_StateManager Component. Once added press the "State Manager" button to open the StateManager Editor Screen. The ABC_StateManager component is required for all entities that will react to ability hits, it runs the effect logic applied by abilities. It can also configure health, hit animations and more detailed in the documentation found by clicking this link: State Manager Config Documentation.

27. On the new editor screen select the Health button to the left, this will load all the health settings ready to be changed. From here you can configure the current health, max health and how much health regens each second. The defaults can be used for this tutorial but you are welcome to change and play with the values.

28. Next is to configure the health GUI to let the player how much health is remaining on the enemy. Select the 'Add Health GUI' button to make a new section appear below. Previously we have used the load default button to add the Slider and Text object, however this is already used by the player so we are going to make new ones. Select the HealthAndMana object in the inspector which is sitting under the ABC_GUIs and copy paste it. Rename this to 'HealthEnemy' and delete the sliderMana. At this point you can move the slider to another point in the scene and change the colours if you wish.

29. Now click and drag the HealthEnemy 'sliderHealth' object to the HealthSlider setting, the 'sliderHealthOverTime' object to the HealthOverTimeSlider setting and the 'txtHealthValue' to the Health Text setting.

30. The last step to finish setting up the enemy is to add hit animations, these will play everytime the fireball hits. Press the 'Hit Animations' button in the menu on the left to load the config screen and hit the 'Add Animation' button to add a hit animation ready to be configured. There is no limit to how many hit animations can be added. From here follow the screenshot below to enter the correct config. This links to the enemies Animator initiating the parameter everytime a hit occurs. It is possible to set specific hit animations that only play from a special 'Hit Animation' effect. In this case it will just run the HitReactionTrigger parameter which will randomly play 1 of many hit animations setup.

In Game

The tutorial has been completed! Run the game and move around with the mouse and WSAD buttons. If you are in target range of the enemy a blue circle will appear at it's feet indicating that it's been targetted. At any time press the F key or the key you selected to activate the fireball spell. Ethan will then prepare the fireball and shoot it at the enemy. If no enemy has been selected it will just shoot the fireball forward due to the Always Travel setting. At this point I recommend playing around with the ability, modify it, try additional settings or even create new abilities.
