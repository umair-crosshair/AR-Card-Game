Effect Event Subscriber

Overview

A script component is avaliable named ABC_EffectEventSubscriber which will allow for you to easily integrate other components when effects are applied and/or removed. When an entity is inflicted by a certain effect you can link that to call other methods in your own scripts. An example is making an ability which when used will apply a custom "interact" effect to near by objects. The nearby objects with the effect subscriber component attached will listen for the "interact effect", running certain methods once the effect is applied. Doing this you can simulate a character opening a door - the character can have the interact effect ability and the door can have the event subscriber.

Setting Up The Subscriber

Follow the below steps to set-up the effect event subscriber:

1. Add the ABC_EffectEventSubscriber to the entity which will listen for effects applied to itself.

2. Change the size of the Custom Effect Events or Custom Effect Removal Events. Each increment allows for an extra custom effect to listen out for.

3. Enter the name of the Effect you want the subscriber to listen for in the "Effect Name" text box

4. Add a Unity Event by pressing the "+" button. The Unity Event we are about to setup here will determine what function(s) to call once the custom effect defined in step 3 is added/removed.

5. Find and select the GameObject which holds the component and the method you wish to invoke when the custom effect is added and or removed. Once the Gameobject is selected drag it into the box below the "Runtime Only" dropdown. This GameObject does not have to be the same entity that has the subscriber component attached, so it is possible to have an entity which just listens out for certain effects which then runs methods on other Objects.

6. Once the GameObject has been inserted the dropdown to the right of the "Runtime Only" dropdown will become avaliable enabling you to selection the function which will run.

7. Setup has been completed. Now the function selected will be called everytime the entity with the ABC_EffectEventSubscriber component is inflicted with an Effect that matches the text in the "Effect Name" box.

Note: For an ability effect to raise the event picked up by the ABC_EffectEventSubscriber the effects Event Type needs to be either "Raise Event" or "Both Simultaniously". Information on changing the effects Event Type can be found by clicking the following link: Event Type

Effect Details

Overview

Below lists all the effects which can be setup on abilities. Effects can be applied to any entities that abilities collide and hit. Entities require the ABC_StateManager Component for effects to activate.

In this section you will find details on what each effect does and how to modify its behaviour and potency through the 3 additional settings (PotencyValue, MiscProperty and AltProperty). For example the Push effect can be modified by adjusting the push distance via the potency value, how long it takes the entity to reach the push distance via the misc property and the lift force via the alt misc property.

This section also details what happens when certain effects are removed from the entity (if setup to do so via the Enable Remove Effect setting). For example the Adjust Stat Value effect will increase a stat value like strength. If the effect is removed the stat value will then revert back to what it was before the effect was applied.

Adjust Health

Will increase or decrease the entities current health. Use this to apply damage to enemies or heal allies

  • Potency Value: Amount to adjust health by (can be a positive or negative value)

Adjust Mana

Will increase or decrease the entities current mana. Use this to recover or reduce mana. Mana can be used to activate abilities

  • Potency Value: Amount to adjust mana by (can be a positive or negative value)

Random Hit Animation

Will activate a random hit animation which has been setup in the entities ABC StateManager Component. More information on setting up hit animations can be found by clicking the following link Setting Up Hit Animations

Hit Animation

Will activate a specfic hit animation which has been setup in the entities ABC StateManager Component. More information on setting up hit animations can be found by clicking the following link Setting Up Hit Animations

  • Misc Property: Name of the hit animation to play. The value entered needs to match the name of the hit animation setup.

Set Stat Value

Will set the stat to the value provided. Stats can be setup in the entities ABC StateManager Component. For example the effect can set the entities strength value by the potency provided. More information on setting up stats can be found by clicking the following link Setting Up Entity Stats

  • Potency Value: Amount to set stat too (can be positive or negative)
  • Misc Property: Name of the stat to change. The value entered needs to match the name of the stat setup in the ABC StateManager Component If no stat matches then nothing will occur in game.

Adjust Stat Value

Will increase or decrease a stat value if it exists. Stats can be setup in the entities ABC StateManager Component. For example the effect can adjust the entities strength value increasing/decreasing it by the potency provided. More information on setting up stats can be found by clicking the following link Setting Up Entity Stats

  • Potency Value: Amount to change stat by (can be positive or negative)
  • Misc Property: Name of the stat to change. The value entered needs to match the name of the stat setup in the ABC StateManager Component If no stat matches then nothing will occur in game.

Removal Effect: Will remove the adjusted stat value. For example if the effect added 5 to the entities strength stat then on removal the strength will be reduced by 5.

Shake Camera

Will shake the camera for the amount duration and speed provided in the additional settings. Use this to add 'juice' to the ability impact.

  • Potency Value: Amount of shake to apply to the camera
  • Misc Property: Duration of the shake
  • Alt Misc Property: Speed of the camera shake.

Shake Object

Will shake the entity hit for the amount provided the shake then decreases over time depending on the decay. Once shake decays to 0 the shake will stop. Use this to add 'juice' to the ability impact.

  • Potency Value: The amount of shake performed
  • Misc Property: The amount of the shake decreased each cycle. Also determines the duration, once shake decays to 0 the shake will stop
  • Alt Misc Property: Delay of the shake object

Switch Color

Will activate a color switch for a duration before reverting back to the entites original color

  • Potency Value: Duration for the color switch before it reverts back
  • Misc Property: Hex code of the color to change to (i.e #880000)
  • Alt Misc Property: (Boolean) if true then the emission color will be changed if enable, else the normal color will change

Bullet Time

Will modify the game speed, either slowing it down or speeding it up for the duration provided

  • Potency Value: Amount to slow down or speed game up. 1 is normal speed, adding a value lower then 1 will slow down the game. Adding a value higher then 1 will speed up the game.
  • Misc Property: Duration of the game speed change

Adjust Max Health

Will adjust the entities max health

  • Potency Value: Amount to increase the max health by (can be positive or negative).
  • Misc Property: (True/False) if true then current health will be restored to full, else it will remain the same and only the max health will be adjusted

Removal Effect: Will remove the adjusted max health value. For example if the effect added 30 to the entities max health then on removal the max health will be reduced by 30.

  • Misc Property: (True/False) if true then current mana will be restored to full, else it will remain the same and only the max mana will be adjusted

Removal Effect: Will remove the adjusted max mana value. For example if the effect added 30 to the entities max mana then on removal the max mana will be reduced by 30.

  • Potency Value: Amount to increase health regen by (can be positive or negative).

Adjust Ability Ammo

Will adjust an abilities ammo count

  • Potency Value: Amount to adjust ammo by (can be positive or negative).
  • Misc Property: ID of the ability which will have its ammo adjusted i.e if 5 is entered then the ability with an ID of 5 will have it's ammo adjusted by this effect.

Adjust Ability Global Prepare Time

Will adjust the global prepare time of all abilities by the percentage value

  • Potency Value: % value to adjust global prepare time by (can be positive or negative). the global prepare time default is 100%, lowering this value will means abilities will take less time to prepare before activating. Increasing this value will result in all abilities having longer prepare times.
    For example if this effect adjusted the global prepare time by -80(%) resulting in a value of 20 (from the 100 default) then all ability prepare times will now only take 20% of their normal times preparing then activating 80% faster. meaning an ability with a 10 second prepare time will now only take 2 seconds.

Removal Effect: Will remove the adjusted ability global prepare time value. For example if the effect added -80% global prepare time then on removal the -80% prepare time will be removed

Adjust Ability Global Cooldown

Will adjust the global cooldown of all abilities by the percentage value

  • Potency Value: % value to adjust global cooldown by (can be positive or negative). the global cooldown default is 100%, lowering this value will speed up the the cooldown for all abilities. Increasing this value will result in all abilities having longer cooldowns.
    For example if this effect adjusted the global cooldown by -80(%) resulting in a value of 20 (from the 100 default) then all global cooldowns will now only take 20% of their normal times cooling down 80% faster. meaning an ability with a 10 second cooldown will now only take 2 seconds.

Removal Effect: Will remove the adjusted ability global cooldown value. For example if the effect added -80% global cooldown then on removal the -80% cooldown will be removed

Adjust Ability Global Initiation Speed

Will adjust the initiation speed of all abilities

  • Potency Value: % value to adjust global initiation speed by (can be positive or negative). the global speed default is 100%, lowering this value will slow down the initiation speed for all abilities, making a sword swing take longer etc. Increasing this value will result in all abilities initiating faster.
    For example if this effect adjusted the global initiation speed by -25(%) resulting in a value of 75 (from the 100 default) then all initiations will be running 25% slower.

Removal Effect: Will remove the adjusted ability global initiation speed value. For example if the effect added -25% global cooldown then on removal the -25% cooldown will be removed

Temp Ability Activation Interval Adjustment

Will temporarily adjust the entities ability activation interval. This adjustment is reset once the interval wait has passed.

  • Potency Value: Value to adjust ability activation interval by (can be positive or negative). The lower the interval the less time the entity has to wait between activating abilities

Add ABC Tag

Will add an ABC Tag to the entity

  • Misc Property: Name of the tag to add.
    Both ABC tags and Unity tags are used by functionality across the system when checking against certain conditions. An example of this is creating an effect that only applies to entities with an “Enemy” tag. There is never the circumstance where an ABC tag will just be checked over the normal Unity tag. Both are checked on every instance.

Removal Effect: Will remove the ABC tag which was added by the effect

Remove ABC Tag

Will remove an ABC Tag from the entity if it exists

  • Misc Property: Name of the tag to remove.
    Both ABC tags and Unity tags are used by functionality across the system when checking against certain conditions. An example of this is creating an effect that only applies to entities with an “Enemy” tag. There is never the circumstance where an ABC tag will just be checked over the normal Unity tag. Both are checked on every instance.

Removal Effect: Will add the ABC tag which was removed by the effect

Toggle Effect Protection

Will enable/disable effect protection, if enabled then the entity is immune to all effects unless the ability has effect protection bypass enabled.

  • Misc Property: (Boolean) if true then effect protection will be enabled, else disabled

Removal Effect: Will enable the effect protection if it was initially disabled or disable the protection if it was initially enabled.

Toggle Ability Activation

Will enable/disable ability activation

  • Misc Property: (Boolean) if true then ability activation will be enabled, else disabled

Removal Effect: Will enable the entities ability activation if it was initially disabled or disable the entities ability activation if it was initially enabled.

Removal Effect: Will enable the entities toggle ability collision initially disabled or disable the entities toggle ability collision if it was initially enabled.
