Weapon Settings Overview

Overview

This part of the system allows for the setup of Weapons. Weapons can be equipped/uneqipped and cycled through during play. Each weapon can be configured to have 1 or more graphics which can then be placed in different positions when equipped and unequipped. For example a Sword graphic can be setup to be in the entities hand when equipped and on his back when unequipped. For equipping/unequipped weapons animations can be added.

Other functionality includes linking weapons to abilities and groups. By doing this you can set to enable/disable abilities/groups when the weapon is equipped. For example during play when the entity equips the sword weapon then all the sword related attacks and abilities will be enabled ready to use whilst all other abilities will be disabled. Weapons can also be assigned to groups which will enable the weapon ready to be equipped when a certain group is activated (i.e in fire mode allow for fire weapons to be equipped)

The last group of settings in this section determines weapon dropping functionality. You can set if the weapon can be dropped and what happens when doing so i.e disable/delete the relating abilities/groups/ui. An animation can also be setup to play when dropping a weapon and a weapon drop object can be added which will drop an object when the weapon is 'dropped' in the system. It's recommended that the weapon drop object has the weapon pick up component which allows for other players to pick up the weapon. More information on this is found by clicking the following link: Weapon Pick Up

Weapon List

At the left of this section is a list of all the weapons which have been setup for the entity. The weapon in the list can be dragged to a different position by clicking the “=” and moving the mouse The weapons will be equipped and cycled through in the order of the list.

Selecting an weapon in the list will load it’s settings to the right. New weapons can be added by hitting the “+ Add New Weapon” button and if a weapon is currently selected it can be copied with the “Copy Selected Weapon” button. Any selected weapon can also be deleted by pressing the “X Delete Selected Weapon” Button.

Once a weapon selected the settings detailed below will appear.

General & Graphic

Overview

Within this sction you can define weapon information, setup graphics and define which key can be triggered to equip the weapon

ID

An automatically generated number which links to the weapon. This is used by the system to always identify the weapon even if the group name changes.

Equip Weapon

Used for in game debugging only. Clicking this tick box will make the entity equip the weapon

Weapon Name

The name of the weapon, this will modify the name in the list and any text can be entered here.

Icon Image

2D texture representing the weapon icon. This icon will be displayed on GUIs during play, including icons; UI Icons setup

Description

A description of the weapon which can be displayed on tooltips when creating Icons.

Weapon Enabled

If ticked then the weapon is enabled and can be equipped, the weapon can be enabled/disabled via effects and other functionality.

Disable Ability Activation On Switch

If ticked then ability activiation will be disabled for a duration when equipping the weapon, allowing the entity to swap weapons with no interruptions. Once clicked the following setting will appear:

  • Disable Duration
      The duration that ability activation will be disabled when equipping the weapon

Equip Duration

How long it takes for the weapon to equip. This duration should be greater then how long it takes for the graphic to show and animation to play out. Lets the system know how long to wait for when equipping the weapon, certain functionality will wait for this time. For example you will not be able to equip the next weapon during this time.

Unequip Duration

How long it takes for the weapon to unequip. This duration should be greater then how long it takes for the graphic to show and animation to play out. Lets the system know how long to wait for when unequipping the weapon, certain functionality will wait for this time. For example you will not be able to equip the next weapon during this time.

Equip On Cycle

If ticked then the weapon can be equipped from being cycled through. Requires the next/previous weapon trigger to be setup. More information on setting up the cycle trigger can be found by clicking the following link: Cycle Next Weapon

Equip On Input

If ticked then the weapon can be set to be equipped from the player pressing a key

If ticked then the following settings will appear:

  • Input Type
      The type of input expected to trigger the equip weapon event, either a hardcoded key can be inserted or a button string which links to the same string setup in the Unity Input Manager.
  • Input Cancel Key (If ‘Key’ Input Type is selected)
      A key can be selected from Unity’s predefined drop down. This key is hardcoded.
  • Input Cancel Button (If ‘Button’ Input Type is selected)
      A string can be entered which should match the same string setup in the Unity Input Manager. This method allows the end user to configure inputs as per Unity functionality.

Weapon Graphics

Weapon graphics can be setup for the weapon. There is no limit to how many graphics can be setup to allow for situations where the entity may dual wield swords or carry 2 guns at once. For each graphic there is settings to define the position where it sits when enabled and disabled. There is also an option to hide the graphic when disabled To aid with animation timings you can also choose the delay before the graphic is shown and placed in it's enable and disable position.

Weapon graphics can be added by clicking the "+ Add Weapon Graphic" button. This will then generate settings for each weapon graphic added. The weapon graphics will be created in the order they appear in the config but the order of the weapon graphic can be changed by clicking the up and down arrows. Weapon graphics can also be deleted by clicking the red X. Clicking the arrow toggle will show the position settings these can then be hidden again by reclicking the arrow toggle. For each graphic you can set the enable and disable position. When a weapon graphic is added the below settings will appear.

Main Graphic

Any object can be added here which will appear when the weapon is equipped. This should be the actual graphic of the weapon (sword, gun, staff etc) This graphic is what will display in the enable/disable position. This graphic is what the entity will be equipping and holding etc.

Sub Graphic

A second object can be added here which will become a child of the Main Graphic described above. This object like the main graphic will display in the weapon enable/disable position

Equip Delay

The delay before the graphic is shown/placed when equipping the weapon, modifying this setting will allow you to time with the animation when the weapon should be placed into it's equip position. I.e after the hand reaches to the weapon etc

UnEquip Delay

The delay before the graphic is hidden/placed when unequipping the weapon, modifying this setting will allow you to time with the animation when the weapon should be placed into it's unequip position. I.e when hand reaches weapon sheath etc

Equip Graphic Position

Determines the position for the weapon when it is equipped. When equipped the graphic it will become a child of the position becoming attached. A number of graphic positions can be selected. The following options are available:

  • On Object
      The weapon graphics position will be at a gameobject defined in additional setting ‘Select Object’. Any object can be dragged or selected onto the setting field. The setting will appear when this option is selected.
      The graphic will become a child of the Object added to the setting.
  • On Self Tag
      The weapon graphics starting position will be at a object which is either the entity object the ABC system is attached too or any child objects linked to the entity which also has a tag which matches the tag defined in additional setting ‘Select Tag’. Only Unity defined tags can be selected from the dropdown. The setting will appear when this option is selected.
      The graphic will become a child of the gameobject found when searching for the self tag.
  • On Tag
      The graphics starting position will be at a gameobject which has a tag which matches the tag defined in additional setting ‘Select Tag’. Only Unity defined tags can be selected from the dropdown. The setting will appear when this option is selected.
      The gameobject with the tag is chosen using ‘FindGameObjectWithTag’ so objects found may vary if more then one exists with the same tag.
      The graphic will become a child of the gameobject found when searching for the tag.
  • Self
      The graphics starting position will be at the entity object the ABC system is attached too
      The graphic will become a child of the entity

Equip Position Offset

X, Y, Z float values can be entered which will modify where the weapon is positioned from it’s equip position. I.e 90 in the X box will move the weapon up the X axis by 90 from the position it appears at.

Equip Forward Offset

A float value can be entered here to move the weapon object forward from it’s equip position. If negative value is entered it will move the weapon object backwards rather then forwards.

Equip Right Offset

A float value can be entered here to move the weapon object to the right from its equip position. If negative value is entered it will move the weapon left rather then right.

Equip Starting Rotation

Determines the starting rotation for the weapon graphic and which way the object is facing. X, Y, Z values can be entered to adjust the weapon graphic starting rotation by the values provided. I.e rotate the weapon around the X axis by 90.

Equip Set Euler Rotation

If enabled then the starting rotation amendments will also be applied to the weapons Local Euler Angles.

Equip Animations

Overview

Within this sction you can define the animations that will run when equipping and unequipping the weapon

Use Animations

If disabled then animations will not activate when equipping/deactivating weapons

Equip Animation Clip

Configuration can be setup to play an animation when the weapon is equipped, the animation will run for the duration defined in the settings. If the animation is shorter than the duration then the animation may repeat.

Animation Clips defined will play on ABC's own Animation Runner system which means you can set to run animations without using Unity's Animator. To determine which animation to play simply select or drag and drop an Animation Clip into the setting. With all animations in ABC you have the option of using one or both of the animation systems (Unity Animator/ABC Animation Runner). It is recommended however that only one is used. Once an Animation Clip is selected the following settings will appear:

  • Animate on Entity
      If ticked then the animation clip will run on the entity that the ABC component is on.
  • Animate on Scroll Ability Graphic
      If ticked then the animation clip will run on the current scroll abilities graphic (if it exists). This setting should be ticked if the current scroll ability graphic object has its own initiating animation you want to play (like gun recoil when it shoots).
  • Animate on Weapon
      If ticked then the animation clip will run on the current weapon graphics (if it exists). This setting should be ticked if the current weapon has its own animation you want to play.
  • Duration
      How long the animation clip will run for before it is stopped
  • Speed
      The speed of the animation clip. 1 is normal speed. A higher value will speed up the animation clip. A lower number will slow the animation clip down.
  • Delay
      Defines the delay before the animation clip is run

Equip Animator Parameter

Configuration can be setup to play an animation when the weapon is equipped, the animation will run for the duration defined in the settings. If the animation is shorter than the duration then the animation may repeat. The animation system links directly into the Unity animation controller attached to the entity and is required for the animations to activate. To determine which animation to play the “Animation Parameter” which has been setup in Unity needs to be written in the parameter config box, then the matching parameter type needs to be selected from the drop down, depending on the parameter selected the following settings will appear with default values:

  • Trigger Parameter
      No further settings will appear as the animation type is a trigger and will just activate once as per Unity system
  • Float, Integer Parameter
      If either of these options are selected then a On Value and Off Value can be entered. The animation will activate the parameter with the On value when the ability starts initiating and then deactivate with the Off value after the duration set. Only a float/integer value should be entered.
  • Bool Parameter
      If either of these options are selected then a On Value and Off Value can be entered. The animation will activate the parameter with the On value when the ability starts initiating and then deactivate with the Off value after the duration set. Only “True” should be entered in the On value and “False” entered in the off value.

Equip Animate on Entity

If ticked then the animation will activate on the animator attached to the entity that the ABC component is on.

Equip Animate on Scroll Ability Graphic

If ticked then the animation will activate on the animator attached to the current equipped weapon graphic (if it exists). This setting should be ticked if the current weapon graphic object has its own animation you want to active.

Equip Animate on Weapon

If ticked then the animation will activate on the animator attached to the current weapon graphic (if it exists). This setting should be ticked if the current weapon graphic object has its own animation you want to active.

Ammo & Reload

Overview

Within this sction you can define how much ammo the weapon has for abilities to consume. You can also set for the weapon to reload including animations and graphics which appear when reloading

Use Ammo

If enabled then the weapon can be setup with ammo which abilities can consume to activate. if an ability is set to use weapon ammo then on activation it will reduce 1 from the weapons ammo count. If the ammo count is 0 then the ability will not activate. There is no limits to how many abilities can consume the weapon ammo To set abilities to use the current equipped weapons ammo click the following link: Use Equipped Weapon Ammo Although named as ammo this can also be used to control how many times an ability can be used in game. Once ticked the following settings appear:

  • Ammo Count
      The current ammo count for the weapon i.e how much ammo the entity has for the ability.
      Ammo count can be adjusted via ability effects. More information on setting up ability effects be found by clicking the following link: Ability Effects Overview

Use Reload

If enabled then the weapons ammo will be divided into ‘clip sizes’ which determines how many times the weapon through abilities can activate before a reload is required. Everytime the clip count reaches 0 the weapon will require a trigger to be pressed to ‘reload’ filling up the clip. When the clip count reaches 0 The abilities using the weapon can’t activate again until the reload has filled the the clip (if possible, if weapon only has 6 ammo remaining and clip size is 10 it will only fill clip up to 6).

The clip can also be reloaded early discarding the current clip count but filling the clip to full ready for an upcoming battle. Although named after gun mechanics this in theory isn’t a setting just for guns. You may create an energy ability which has 100 shots and after 10 ‘shots’ (clip size being 10) the entity will need to recharge his energy with a click of a button.

Reloading will be cancelled If the weapon is disabled/unequipped due to the entity swapping to another weapon. Once the weapon is re-equipped/re-enabled then it can once again be reloaded.

Example of use: Weapon has an ammo count of 100 and a clip size of 10. After 10 shots used by an ability, the weapon will require a trigger to be pressed to reload, filling the clip back up to 10 and reducing the ammo count by 10.

Animations and Graphics which activate on reload can be setup with the settings defined below. The reload trigger is configured in the ABC Controller settings. More information on setting up the reload trigger can be found by clicking the following link: Reload Input Type

When ticked the following settings will appear:

  • Clip Size
      The size of the ‘Clip’ explained in more detail above. Determines how many times the weapon ammo can be used before a reload is required.
  • Reload Duration
      How long it takes to reload the weapon, after the duration the ammo/clip values will change.
  • Reload Fill Clip
      If enabled then the clip count will increase one by one with the interval being the reload duration. If disabled then normal functionality will occur of the whole clip filling to the max possible after the reload duration defined.
      Can be interrupted with the activation of the ability, i.e can reload one by one and stop halfway to the clip being full to activate.
      Useful for weapons where the entity will be filling ammo one by one like shotgun shells.
  • Fill Clip Repeat Graphic
      Will appear if “Reload Fill Clip” is ticked
      If enabled then any reloading graphics setup will repeat each time the clip is being refilled (one by one).
  • Reload Restrict Ability activation Duration
      Defines how long the entities ability activation will be restricted for when the reload has started.
  • Auto Reload When Required
      Will automatically trigger a reload on the weapon when the clip reaches 0

Abilities & Groups

Overview

Within this sction you can link weapon to abilities to have the abilities either enable or disable when the weapon is equipped. You can also link the weapon to groups so the weapon is enabled when a group is enabled. On the flip side you can also enable or disable groups when the weapon is equipped. All these options allow you to control in game which abilities can be used when weapons are equipped. It is good practice to only have abilities enabled which relate to the weapon disabling the rest. For groups you can set it up so certain weapons are only avaliable when a group is activated like a special fire mode etc

Enable Abilities When Equipped

If ticked then abilities can be set to be enabled once the weapon is equipped. Once enabled the following settings will appear:

  • Disable All Other Abilities
      If enabled then any ability not listed below will be disabled when the weapon is equipped.
  • Adding Abilities To Enable
      To add abilities which will enable when the weapon is equipped simply find the ability in the dropdown list, select it and press the "+" button Once done it will get added to the list of abilities to enable, there is no limit to the amount of abilities which can be enabled. Abilities can be removed from the list by pressing the 'x' button. You can also reorder the list. Abilities will be enabled in the order of the list.

Disable Abilities When Equipped

If ticked then abilities can be set to be disabled once the weapon is equipped. Once enabled the following settings will appear:

  • Enable All Other Abilities
      If enabled then any ability not listed below will be enabled when the weapon is equipped.
  • Adding Abilities To Disable
      To add abilities which will disable when the weapon is equipped simply find the ability in the dropdown list, select it and press the "+" button Once done it will get added to the list of abilities to disable, there is no limit to the amount of abilities which can be disabled. Abilities can be removed from the list by pressing the 'x' button. You can also reorder the list. Abilities will be disabled in the order of the list.

Allow Group Assignment

If enabled then the weapon can be assigned to an ABC Group which has been setup. Grouped weapons can be set to become active all at the same time after different events. More details on Groups and how to set them up can be found by clicking the following link: Group Information

If ticked then further config will appear to assign the weapon to a group. The weapon can be added to a group by selecting a group in the dropdown next to the “Assign To Group” label and then pressing the “+” button. The weapon can be removed from a group by hitting the “X” button at the end of the row of the group you no longer want the weapon to be assigned too.

Enable Groups When Equipped

If ticked then groups can be set to be enabled once the weapon is equipped. Once enabled the following settings will appear:

  • Disable All Other Groups
      If enabled then any groups not listed below will be disabled when the weapon is equipped.
  • Adding Groups To Enable
      To add groups which will enable when the weapon is equipped simply find the group in the dropdown list, select it and press the "+" button Once done it will get added to the list of groups to enable, there is no limit to the amount of groups which can be enabled. Groups can be removed from the list by pressing the 'x' button. You can also reorder the list. Groups will be enabled in the order of the list.

Disable Groups When Equipped

If ticked then groups can be set to be disabled once the weapon is equipped. Once enabled the following settings will appear:

  • Enable All Other Groups
      If enabled then any groups not listed below will be enabled when the weapon is equipped.
  • Adding Groups To Disable
      To add groups which will disable when the weapon is equipped simply find the group in the dropdown list, select it and press the "+" button Once done it will get added to the list of groups to disable, there is no limit to the amount of groups which can be disabled. Groups can be removed from the list by pressing the 'x' button. You can also reorder the list. Groups will be disabled in the order of the list.

Weapon Drop

Overview

Within this sction you can define settings which will allow for your entity to drop the weapon. When dropped you can choose for the weapon to be disabled or deleted. You can also set for the linking enable abilities, groups and UI to also be disabled/deleted. Animations can run when dropping the weapon and you can define for a weapon drop object to appear in game. The weapon drop object can be any GameObject but it's recommended to have an object which has the ABC_WeaponPickUp component attached if you want any entities to pick up the weapon again. More information on the pickup component can be found by clicking the following link: Weapon Pick Up

Enable Weapon Drop

If ticked then the weapon is droppable and further settings will appear

Weapon Drop Action

Determines the weapon drop action. The action select will be applied to the weapon, linked enable abilities, groups and UI. Either Disable or Delete can be selected from the dropdown. Disable will simply disable the weapon, abilities etc. Delete will remove them completely from the entity. The only way to import them back after is via weapon pick up if set to import weapons/abilities/groups etc.

Drop Duration

Sets how long it takes for the weapon to drop, this should be higher then any animations etc and lets ABC know how long to wait before equipping another weapon after the drop

Apply Action To Weapon Enable Abilities

If ticked then the weapon drop action (Disable or Delete) will also be applied to the abilities that are enabled when this weapon is equipped

Apply Action To Weapon Assigned Groups

If ticked then the weapon drop action (Disable or Delete) will also be applied to the groups the weapon is assigned too

Apply Action To Weapon Assigned UI

If ticked then the weapon drop action (Disable or Delete) will also be applied to the UI the weapon is assigned too

The animation system links directly into the Unity animation controller attached to the entity and is required for the animations to activate. To determine which animation to play the “Animation Parameter” which has been setup in Unity needs to be written in the parameter config box, then the matching parameter type needs to be selected from the drop down, depending on the parameter selected the following settings will appear with default values:

  • Trigger Parameter
      No further settings will appear as the animation type is a trigger and will just activate once as per Unity system
  • Float, Integer Parameter
      If either of these options are selected then a On Value and Off Value can be entered. The animation will activate the parameter with the On value when the ability starts initiating and then deactivate with the Off value after the duration set. Only a float/integer value should be entered.
  • Bool Parameter
      If either of these options are selected then a On Value and Off Value can be entered. The animation will activate the parameter with the On value when the ability starts initiating and then deactivate with the Off value after the duration set. Only “True” should be entered in the On value and “False” entered in the off value.

Use Weapon Drop Object

If enabled then a gameobject will be dropped during play when the weapon is dropped

Drop Object

Object to drop when weapon is dropped, can be any gameobject but it is recommend that you add and configure the ABC_WeaponPickUp component if you wish for players to pick up the weapon again. More information on the weapon pick up component can be found here: Weapon Pick Up

Object Drop Delay

The delay before the weapon drop object appears in game

Object Drop Duration

The duration the drop object will remain in game, if set to 0 then the duration is unlimited

Update Weapon Drop Ammo

If true then the weapon drop object will have it's pick up ammo updated to match the current ammo on the weapon, so when someone else picks up the weapon it will have the ammo which the weapon dropped had
