Settings

Overview

Once the ability has been selected then the abilities general settings can be configured including ability type, name, description, graphic, mana cost, cooldown, prepare time and more

Name

The name of the ability is entered here. The name will be displayed in logs during play and through lists and dropdowns in the editor windows.

Icon Image

2D texture representing the ability icon. This icon will be displayed in the top left of the editor when the ability is selected and on GUIs during play, including; UI Icons setup and the current enabled scroll ability GUI Icon

Main Graphic

Main graphic object which represents the ability during play Setting will not show if Ability Type is set too Raycast. Any gameobject can be set which will be instantiated in game when the ability is activated. There is no restriction to what gameobject can be added. For example a model of a missile or a particle effect of a fireball can be set. If no object is set then ABC will revert to the default projectile particle during play.

Sub Graphic

A second object can be added here which will become a child of the Main Graphic described above. Setting will not show if Ability Type is set too Raycast.This object like the main graphic will be instantiated in game when the ability is activated.

Type

The type of ability. Depending on the option selected different settings will appear across the ability manager. Either Particle Effect, Melee or Raycast can be selected which represent the following:

  • Particle Effect
      When activated will create a gameobject in game which can travel and collide with objects applying effects. Will become its own entity by not becoming a child of any other object allowing it to move on it’s own accord.
  • RayCast
      When activated will raycast out via to different target points depending on the travel type selected. Any objects hit will have effects applied. No gameobject is created. Recommend for FPS type games
  • Melee
      When activated will create a gameobject which will become a child of the starting position object, allowing for animations to control the movement of the ability (like a sword swipe). This type of ability has different settings and functionality typically used for melee combat. Will collide with objects applying effects.
      If selected then the abilities Travel Type will default to “No Travel”

Scroll Ability

If ticked then the ability will be flagged as a Scroll Ability. Only one scrollable ability can be enabled at once during play and all scroll abilities will trigger with the same key/button. For example during play the entity will ‘equip’/enable one of the scroll abilities which can be activated with a key/button. The entity can then switch to another scroll ability which is next in the list ‘equipping/enabling it’ and disabling the scroll ability it switched from. The newly equipped/enabled scroll ability can then be activated using the same key/button as before.

This can be used in a game where you want to cycle weapons like an FPS. All scrollable abilities will activate via one trigger. Configuration to setup what key will activate the current equipped scroll ability and cycle to the next and previous scroll ability can be done in the ABC Controller settings. More information on setting up ability scroll triggers can be found by clicking the following link: Scrollable & Reload Triggers

Setup can also be done to enable a group of abilities once a specific scroll ability becomes enabled/’equipped’. This can be used to setup a weapon system where equipping a hammer will enable certain melee abilities, whilst then switching to a sword will enable other melee abilities etc. More information on setting up ability groups can be found by clicking the following link: Ability Groups

Different settings throughout the ability manager will appear if the ability is set as a Scroll Ability.

Can Activate During Hit Prevention

Ticking this box will allow the ability to start activating even if the entities ability activation has been prevented due to a recent hit. The Setting to enable/disable if the entities ability activation can be restricted from a hit can be found in the ABC Controller Settings, for more information click the following link: Hit Can Prevent Activation Setting

Hit Prevention Wont Interrupt Activation

Ticking this box will ensure that the abilities activation will never be interrupted due to a recent hit. Setting to enable/disable if the entities abilities activation can be interrupted from a hit can be found in the ABC Controller Settings, for more information click the following link: Hit Can Interrupt Activation Setting

Ability Toggle

Determines if the ability can be toggled on/off. If the ability toggle is on then the ability will activate and stay in play until the duration is up or it has been toggled off. By default this is turned off so the ability will stay in play until its duration is up and will never end early due to being toggled off.

An example for this is if you want to hold down a key to activate a shield ability which will disappear when the player lets go of the key. 3 Options can be selected from the drop down: Off, Hold and On Off

  • Off
      Ability toggle is disabled allowing the ability to act as normal. Nothing will be able to toggle it off
  • Hold
      Ability will remain in play for as long as it’s key/button trigger is held down unless it ends through other functionality like destroying on collision or having a duration. Whilst the ability is active if the key/button is released the ability will end.
  • On Off
      Pressing the abilities key/button trigger once will activate the ability. Ability will remain in play until the same key/button trigger is pressed again. Ability can also end early through other functionality like destroying on collision or having a duration.

If the Ability Toggle is set to “Hold” or “On Off”. the following setting will appear:

  • Can Activate Abilities When Toggled
      If enabled then the entity can activate other abilities whilst this ability is toggled on. If false then the entity can not activate other abilities whilst this ability active during play.

If Can Activate Abilities When Toggled setting is disabled the following setting will appear:

  • Repeat Initiating Animation Whilst Toggled
      If enabled then the Initiating Animation will be repeated whilst the ability is toggled on during play. The animation will stop repeating once the ability is toggled off

Land Or Air

A dropdown which determines the type of elevation required by the entity to activate the ability, i.e only activate ability if the entity is on the ground and not in the air. The following dropdown options can be selected:

  • Land Or Air
      Set as default, if this option is selected then the ability will not take elevation into account. Entity can be in the air or on the ground it won’t matter as ability will ignore the elevation status and always activate.
  • Land
      Ability can only activate if the entity is on the ground and not currently in the air. System will consider the entity on ‘Land’ if the entity is less than 1 distance from the ground.
  • Air
      Ability can only activate if the entity is in the air (not on the ground). If selected a second field will appear “Distance From Ground” which will determine how far the entity needs to be from the ground to be considered as in the air. e.g if 4 is set then the entity will be in the air if the distance between the below object and the entity is greater than 4.

Force Activation

If enabled then the ability will activate even if the entity is currently restricted from activating abilities. This will override:

  • Ability activation intervals (the interval between abilities being allowed to activate)
  • Idle mode restrictions
  • Restrictions due to other abilities currently being activated (note: Will allow the forced ability to activate even if another was also being activated. If both abilities have animations setup then these would need to be accurately timed not to conflict with each other.)
  • Toggled abilities activated in play (toggled abilities can be setup so once they are in play no other abilities can be activated)
  • Ability activation restricted due to an ability reloading

Hits which have restricted the entity from activating abilities are not taken into consideration due to another setting “Can Activate During Hit Prevention” which determines if the ability can activate when the entity is ‘hit restricted’.

If ticked enabled the following setting will appear:

  • Interrupt Current Activation
      If enabled then any abilities currently being activated will be interrupted when the force ability is activated.
      An example for this is having a ‘Dodge’ ability which will interrupt a melee ability currently activating, which will stop the entity from attacking to dodge away.

Prepare Time

Determines how long the ability will wait before initiating, simulates the entity 'casting/activating/preparing'. If 0 is set then the ability will skip the preparation stage. If the ability is set to prepare then additional settings will become available throughout like animations which can play out whilst the ability is preparing (i.e simulate the entity casting/activating/using the ability).

Show on GUI

If the Preparing Slider or Preparing Text has been setup in the ABC Controller Settings then ticking this setting will show the current preparation progress of the ability on the slider/text configured. Once the slider reaches the end the ability will initiate.

If left unticked then the slider or text GUI setup will not be used whilst the ability is preparing.

Ignore Prepare Time Adjustments

If ticked then the prepare time can not be modified during play for example by effects which increase/decrease prepare time. The prepare time will always be what was set.

Moving Interrupts Preparing

If true then moving the entity from its original activation position can interrupt the preparation, stopping the ability from activating. The system will record the position that the entity started preparing the ability. If the entity moves a defined distance away from the starting position then the abilities activation will be interrupted.

If enabled the following setting will appear:

  • Distance to Interrupt
      Defines how far away the entity needs to move from the starting position to interrupt the preparation, stopping the ability from activating.

Duration

Determines how long the ability will remain in play before it is automatically disabled. Ability can be disabled before the duration is up through other functionality like being setup to destroy due to object collision. If 0 is set then the ability will have an unlimited duration.

Travel Speed

Determines how fast the ability will travel. Setting will only show if the Ability Type is set to “Particle Effect”

Raycast Length

When the Ability Type is set to "Raycast" then on activation a raycast will be sent out, inflicting effects on all those hit. This setting determines the length the Raycast will go.

Affect Layer

When the Ability Type is set to "Raycast" then on activation a raycast will be sent out, inflicting effects on all those hit. This setting determines what Unity Layer the raycast will be on (only objects on the same layer as the raycast will hit).

Raycast Radius

When the Ability Type is set to "Raycast" then on activation a raycast will be sent out, inflicting effects on all those hit. This setting determines how wide the raycast radius is. The higher the radius the bigger the height and width of the cast hitting more entities.

A wider width would be appropriate to use for a shotgun ability as this general hits in a cone shape and not just a straight line.

Raycast Hit Max

When the Ability Type is set to "Raycast" then on activation a raycast will be sent out, inflicting effects on all those hit. This setting determines how many objects will be ‘hit’ before the ability stops. I.e if 2 is set then the ability will inflict effects on the first 2 entities hit by the raycast and then stop. If 0 is set then there is no limit to how many can be hit.

Raycast Blockable

When the Ability Type is set to "Raycast" then on activation a raycast will be sent out, inflicting effects on all those hit. If this setting is enabled then the raycast can be blocked by non ABC StateManager objects, stopping the cast in its path and stopping anything behind the blocked object from being hit. If disabled the raycast will travel out ignoring all objects that do not have the ABC StateManager component.

Raycast Ignore Terrain

When the Ability Type is set to "Raycast" then on activation a raycast will be sent out, inflicting effects on all those hit. If this setting is enabled then the raycast will ignore the active terrain disregarding it as an object hit by the raycast.

Mana Cost

Determines how much mana is required to activate the ability. If the entity does not have enough mana then the ability will not activate.

Reduce Mana Whilst Active

If enabled then whilst the ability is active in play the entities mana will be reduced by the cost every second. If the entities mana reaches 0 the ability will be destroyed. By ticking this box and setting the ability to be toggled on/off with an infinite duration you can make abilities which will drain mana whilst toggled on, like a shield.

Cooldown

Determines how long the entity will have to wait after the ability activates to activate it again. This links into the Cooldown UI Icon settings which can be setup to represent on an icon bar when the ability is ready to activate again after it was activated. More information on setting up cooldown icon GUIs can be found by clicking the following link: UI Icons cooldown setup

Start Cooldown After Ability Ends

If enabled then the cooldown described above will not start until the ability which was just activated is destroyed. I.e if a fireball is activated then the recast won’t start until the fireball is destroyed.

Use Ammo

If enabled then the ability will require ammo to activate. Each time the ability activates it will reduce 1 from the abilities ammo count. If the ammo count is 0 then the ability will not activate. Although named as ammo this can also be used to control how many times an ability can be used in game. Once ticked the following settings appear:

  • Ammo Count
      The current ammo count for the ability i.e how much ammo the entity has for the ability.
      Ammo count can be adjusted via ability effects. More information on setting up ability effects be found by clicking the following link: Ability Effects Overview
  • Use Equipped Weapon Ammo
      If ticked then the ability will consume ammo from the current equipped weapon. You can control which abilities are enabled/disabled when a weapon is equipped in the following settings described here: Enable Abilities When Weapon Equipped This allows you to define which ability takes from what weapon ammo. If ticked then ammo for the ability can no longer be used.
      Ammo count can be adjusted via ability effects. More information on setting up ability effects be found by clicking the following link: Ability Effects Overview
  • Reduce Ammo Whilst Active
      If ticked then the ammo count will be reduced by one for each second the ability is active during play
      Setting will not appear if Ability Type “Ray Cast” has been selected

Use Reload

Only useable If ability has been set up as a “Scroll Ability”. More information on scroll abilities can be found by clicking the following link: Scroll Abilities Overview. If enabled then the abilities ammo will be divided into ‘clip sizes’ which determines how many times the ability can activate before a reload is required. Everytime the clip count reaches 0 the ability will require a trigger to be pressed to ‘reload’ filling up the clip. When the clip count reaches 0 The ability can’t be activated again until the reload has filled the the clip (if possible, if ability only has 6 ammo remaining and clip size is 10 it will only fill clip up to 6).

The clip can also be reloaded early discarding the current clip count but filling the clip to full ready for an upcoming battle. Although named after gun mechanics this in theory isn’t a setting just for guns. You may create an energy ability which has 100 shots and after 10 ‘shots’ (clip size being 10) the entity will need to recharge his energy with a click of a button.

Reloading will be cancelled If the scroll ability is disabled/unequipped due to the entity swapping to another scroll ability. Once the scroll ability is re-equipped/re-enabled then it can once again be reloaded.

Example of use: Ability has an ammo count of 100 and a clip size of 10. After 10 shots, the ability will require a trigger to be pressed to reload, filling the clip back up to 10 and reducing the ammo count by 10.

Reload Animations and Graphics can be setup to initiate when reloading is done. The reload trigger is configured in the ABC Controller settings. More information on setting up the reload trigger can be found by clicking the following link: Reload Input Type

When ticked the following settings will appear:

  • Clip Size
      The size of the ‘Clip’ explained in more detail above. Determines how many times the ability can activate before a reload is required.
  • Reload Duration
      How long it takes to reload the ability, after the duration the ammo/clip values will change.
  • Auto Reload When Required
      Will automatically trigger a reload on the ability when the clip reaches 0
  • Reload Fill Clip
      If enabled then the clip count will increase one by one with the interval being the reload duration. If disabled then normal functionality will occur of the whole clip filling to the max possible after the reload duration defined.
      Can be interrupted with the activating of the ability, i.e can reload one by one and stop halfway to the clip being full to activate.
      Useful for abilities where the entity will be filling ammo one by one like shotgun shells.
  • Fill Clip Repeat Graphic
      Will appear if “Reload Fill Clip” is ticked
      If enabled then any reloading graphics setup will repeat each time the clip is being refilled (one by one).
  • Reload Restrict Ability activation Duration
      Defines how long the entities ability activation will be restricted for once the reload has started.

Allow Group Assignment

If enabled then the ability can be assigned to an Ability Group which has been setup. Grouped ability can be set to become active all at the same time after different events. More details on Ability Groups and how to set them up can be found by clicking the following link: Ability Group Information

If ticked then further config will appear to assign the ability to a group. The ability can be added to a group by selecting a group in the dropdown next to the “Assign To Group” label and then pressing the “+” button. The ability can be removed from a group by hitting the “X” button at the end of the row of the group you no longer want the ability to be assigned too.

Key & Combo

Overview

The key and combo section allows you to assign input triggers to activate abilities. Ability combos can also be configured which will allow a chain of abilities to activate one after another, either freely or if the previous combo ability hits an entity. There is also additional settings that will allow for 2 abilities to be linked so both will activate at the same time to make unique combinations.

Trigger Type

A dropdown selection of either 'Input' or 'Input Combo' can be entered. Depending on the selection the ability can trigger from one or two ways:

  • Input
      If Input is selected the ability will trigger from a single key press - once selected additional fields will be displayed to allow for you to select the key or button required to trigger the ability. Additional key setting will also appear in the circumstance you want 2 keys to trigger the ability for example trigger the ability when Shift is held down and the mouse is clicked (for like a dash attack)
  • Input Combo
      If selected then a combination of keys can be setup to trigger the ability i.e F > F > B if inputted will trigger the special attack. Once selected then an additional list box will appear below allowing you to configure which key combination needs to be pressed to trigger the ability. Pressing the '+' will add another key which is required to be pressed. Pressing the up and down arrows will change the order of the inputs. The order from top to bottom determines the order the keys need to be pressed to trigger the ability. The recorded inputs will recycle by an interval by a global setting described in the following link: Input Combo Recycle Interval For example if the interval is set to 1, if F > F is entered then the player has 1 second to enter the B to do the special attack, if done after the interval the ability will not trigger and the player has to enter in again F > F > B.

On Key Press

If trigger type Input is selected then this setting if enabled will allow the ability to activate from a key press trigger. The ability will only be triggered to activate once per press, holding down the key will not initiate a repeated activation attempt.

On Key Down

If trigger type Input is selected then this setting if enabled will allow the ability to repeatedly trigger to activate as long as the key press is held down. This can be used for abilities you want to fire in quick succession i.e a machine gun, however consideration needs to be taken into account on any restrictions delaying the ability from activating rapidly like the Cooldown and Activation Interval

Input Type

If trigger type Input is selected then this setting will appear to select The type of input expected to activate the ability, either a hardcoded key can be inserted or a button string which links to the same string setup in the Unity Input Manager.

  • Key (If ‘Key’ Input Type is selected)
      A key can be selected from Unity’s predefined drop down to trigger the click to activate the ability group. This key is hardcoded.
  • Button (If ‘Button’ Input Type is selected)
      A string can be entered which should match the same string setup in the Unity Input Manager. This method allows the end user to configure inputs as per Unity functionality to activate the ability.

Additional Input

If trigger type Input is selected then this setting will appear. If enabled then the ability will require an additional trigger to activate i.e ability will activate if Right mouse is held down and ‘F’ is clicked etc. This can be setup so the same key triggers will activate different abilities depending on another key held down or pressed at the same time. If enabled the following settings appear:

  • On Key Press
      If enabled then the ability will require an additional key press trigger to activate. The ability will only be triggered to activate once per press, holding down the key will not initiate a repeated activation attempt.
  • On Key Down
      If enabled then the ability will require an additional key press to activate which can be held down to repeatedly trigger the ability activation.
  • Input Type
      The type of additional input expected to activate the ability, either a hardcoded key can be inserted or a button string which links to the same string setup in the Unity Input Manager.
  • Additional Key (If ‘Key’ Input Type is selected)
      A key can be selected from Unity’s predefined drop down to trigger the additional click required to activate the ability. This key is hardcoded.
  • Additional Button (If ‘Button’ Input Type is selected)
      A string can be entered which should match the same string setup in the Unity Input Manager. This method allows the end user to configure inputs as per Unity functionality to trigger the additional click required to activate the ability.

Input Type (Scroll Ability)

The type of input expected to switch/equip/enable the scroll ability, either a hardcoded key can be inserted or a button string which links to the same string setup in the Unity Input Manager. An example for this use is by allowing users to cycle through each scroll ability one by one with the next/previous trigger but also allowing the user to quickly equip a gun by pressing a key setup here like 1.

  • Key (If ‘Key’ Input Type is selected)
      A key can be selected from Unity’s predefined drop down to trigger the click to switch/equip/enable the scroll ability. This key is hardcoded.
  • Button (If ‘Button’ Input Type is selected)
      A string can be entered which should match the same string setup in the Unity Input Manager. This method allows the end user to configure inputs as per Unity functionality to trigger the click to switch/equip/enable the scroll ability.

Swap Duration (Scroll Ability)

How long it takes to switch/equip/enable the scroll ability, no abilities can be activated for this duration. This duration is taken into account when cycling through scroll abilities with the next and previous triggers and through the switch directly to scroll ability trigger explained above.

Set/Unset Raise Event

If true then an event will be raised when the Scroll ability is set and unset informing subscribers that a scroll ability has been initialised/equipped/enabled or uninitialised/uneqipped/disabled. This will be called every time a scroll ability is cycled too or directly switched too.

The event will pass through the ID and name of the scroll ability and a boolean indicating if it was Set or Unset (True if set, else false if unset)

Code example to subscribe to the event delegate: 

                    
                             private void OnEnable() {
                                    this.ScrollAbilitiesABC = transform.GetComponentInChildren<ABC_Controller>();

                                    //subscribe to the event 
                                    if (this.ScrollAbilitiesABC != null)
                                       this.ScrollAbilitiesABC.onScrollAbilitySetAndUnset += this.ScrollAbilitySetEvent;   
                                }

                               public void ScrollAbilitySetEvent(int AbilityID, string AbilityName, bool Set) {
                                     Debug.Log(“ID ” + AbilityID + “ Name “ + AbilityName + “Set Status: “ + Set);    
                                }

                                private void OnDisable() {
                                    //Unsubscribe to any events
                                    if (this.ScrollAbilitiesABC != null)
                                    this.ScrollAbilitiesABC.onScrollAbilitySetAndUnset -= this.ScrollAbilitySetEvent;
                                }

Wait For Key To Initiate

After the ability preparation stage if this setting is enabled then the ability will wait for a second key/button trigger before initiating. No other abilities can be used during this time. The ability is still able to be interrupted before the wait key is triggered.

An example for this setting is activating an ability which charges a massive energy ball shot at the preparation stage. Once it’s done preparing and is fully charged the entity can be move around until the player presses the “Wait For Key To Initiate” trigger, letting off the charged ability at enemies.

If enabled the following settings will appear:

  • Input Type
      The type of input the ability will wait for to move from the preparation to the initiation stage, either a hardcoded key can be inserted or a button string which links to the same string setup in the Unity Input Manager.
  • Key (If ‘Key’ Input Type is selected)
      A key can be selected from Unity’s predefined drop down which the ability will wait for to move from the preparation to the initiation stage. This key is hardcoded.
  • Button (If ‘Button’ Input Type is selected)
      A string can be entered which should match the same string setup in the Unity Input Manager. This method allows the end user to configure inputs as per Unity functionality which the ability will wait for to move from the preparation to the initiation stage.
  • Delay Till Recognised
      Example of use: After preparation if the ability is waiting for “H” to be pressed to move on to the initiation stage then it won’t recognise the “H” press until the delay has passed.

Require Override Crosshair

Ability will only activate if Override Crosshair has been triggered and is present on screen. Configuration to setup the Override Crosshair can be found by clicking the following link: Crosshair Settings

Enable Ability Trigger Links

This setting allows for other abilities to be trigger linked so they will always have the same activation key/button trigger as the ability being configured. Any ability linked will always have the same activation trigger even if the trigger is changed during play.

For example if you had 1 ability “Fireball” and another ability “Fireblast” - which has a trigger link to Fireball. Everytime Fireball key/button activation trigger is changed then Fireblast key/button activation trigger will always match.

Abilities can be added to the trigger link list by selecting an ability in the dropdown next to the “Trigger Links” label and then pressing the “+” button. The ability can be removed from the trigger link list by hitting the “X” button at the end of the row of each ability. Abilities can be ordered using the up and down arrows at the end of the row of each ability. If trigger key/buttons are changed on the ability then it’s trigger linked abilities will have it’s key/buttons changed to match in order of the list.

Ability Combo

If ticked then the ability will be marked as a combo ability. Combo abilities are grouped by the same key/button activation trigger and ordered by the position that they appear in the entities Ability List.

The abilities will activate one after another in order as long as the user keeps pressing the key/button trigger before the “Combo Next Time” duration is up. If the duration is up then the combo chain will restart, starting with the first ability combo in the list. If the last ability in the combo list is activated then naturally the list will restart from the beginning after it’s “Combo Next Time” duration is up.

To set this up simply have 2 abilities marked as a combo with the same activation key/button trigger. The position they appear in the list is the order of the combo. Once the first ability is activated then the user has the “Combo Next Time” duration to hit the trigger again to activate the second ability, if the trigger is hit after the “Combo Next Time” duration then the combo will restart and activate the first ability instead. There is no max to how many abilities can be in a combo.

If an ability with the same activation trigger but not marked as a combo or an ability with a different activation trigger is activated mid combo then the combo chain will break and reset, starting from the first ability on next activation, unless the ability with the different activation trigger has “Never Reset Other Combos” enabled. A setting can also be enabled to only continue the combo if the previous combo ability hit an entity.

Combo Next Time

The amount of time the user has to hit the activation key/button trigger to activate the next ability in the combo chain. If the user hits the key/button trigger after this time the combo is reset. e.g Once the ability is activated then the user has the “Combo Next Time” duration to hit the trigger again to activate the next combo ability, if the trigger is hit after the “Combo Next Time” duration then the combo will restart.

If the ability is last in the combo chain then it will act as the wait time before the combo is reset enabling the first ability to activate again, restarting the combo. If you want the entity to restart the combo quickly after just finishing the previous combo then keep this setting low for the last ability so it will reset the combo quicker.

Combo Hit Required

Ability will only activate if Override Crosshair has been triggered and is present on screen. Configuration to setup the Override Crosshair can be found by clicking the following link: Crosshair Settings

Never Reset Other Combos

If enabled then activating this ability will not reset any combo chains currently in progress. This can be enabled even if the ability is not marked as an “Ability Combo”. If disabled then activating this ability will reset all combos, unless this ability is part of a combo in progress.

Randomly Swap Ability Position

If enabled then the system will randomly every few seconds mix up abilities in the same activation trigger group which also have this setting ticked. This also requires the global setting “Enable Ability Random Position Swap” to be enabled in the ABC Controller Settings. More information on globally setting the enable ability random position swap can be found by clicking the following link: Enable Ability Random Position Swap

Activation Interval Temporary Adjustment

On activation of the ability this setting will temporarily modify the global wait between ability activations, more details on configuring the interval between ability activations can be found here: Activation Interval A positive or negative float value can be entered. The adjustment is reset after the activation interval has passed, so is a one time adjustment.

This can be used to modify how quickly the user can activate another ability after this ability is used, this may be for reasons like the ability requires a lot of physical energy which takes the entity time to recover from meaning the entity is not yet ready to activate abilities at the normal rate. If 0 (the default) is entered then no adjustments will be made.

Enabled Duration

If the duration entered is above 0 then the ability once enabled will become disabled after the time entered. The ability can be enabled via different methods across the system like from ability effects, ability group activation or through enabled events detailed below. Useful for situations like if the ability is enabled from a pickup which only lasts for a limited time.

Enable After Event

If enabled then 2 different events can be setup. When the events occur in game the ability will become enabled. The 2 different events are abilities activation and abilities collision. I.e if a defined ability activates or a defined ability collides then enable this ability. Once ticked 2 list boxes will appear:

  • Enable on - Ability Activation
      Abilities can be added to the enable on activation list by selecting an ability in the dropdown next to the “Ability Activated” label and then pressing the “+” button. The ability can be removed from the list by hitting the “X” button at the end of the row. Abilities can be ordered using the up and down arrows at the end of the row. If any of the abilities in the list activate then the ability being configured will be enabled.
  • Enable on - Ability Collision
      Abilities can be added to the enable on collision list by selecting an ability in the dropdown next to the “Ability Collision” label and then pressing the “+” button. The ability can be removed from the list by hitting the “X” button at the end of the row. Abilities can be ordered using the up and down arrows at the end of the row. If any of the abilities in the list collide/hit an entity during play then the ability being configured will be enabled.

Enable Ability Activation Links

This configuration will allow for the ability to set up activation links. Any abilities linked will activate at the same time as the master ability (which is currently being configured). If the initial master ability does not activate due to restrictions then the linked abilities will also not activate. If one linked ability can’t activate this will not stop the activation of the master ability or other linked abilities. The linked abilities on activation will ignore:

  • Trigger checks (doesn’t matter if the activation key/button trigger was not pressed)
  • Combo checks (it will activate even if it’s part of a combo chain)
  • Ability activation intervals (the interval between abilities being allowed to activate)
  • Idle mode restrictions
  • Restrictions due to other abilities currently being activated (note: Will allow the forced ability to activate even if another was also being activated. If both abilities have animations setup then these would need to be accurately timed not to conflict with each other.)
  • Toggled abilities activated in play (toggled abilities can be setup so once they are in play no other abilities can be activated)
  • Ability activation restricted due to an ability reloading

However all other restrictions will still apply which can stop linked abilities from activating i.e if the entity doesn’t have enough mana or the linked ability requires a target which the entity does not have.

All linked abilities will activate as if they were activated through normal means using all settings configured for them, if linked abilities have different animations setup this may cause a few unexpected results so it’s recommended all animations are setup on the master ability unless timings are done right.

Ability Activation Links provides a whole range of different possibilities which can make abilities unique like sending off multiple projects at once.

You can also play with different settings like preparation times which will mean that abilities don’t appear in game at the same time but one after another etc. Linked abilities can have their own linked abilities and so fourth.

Once enabled the Activation Link list box will appear. Abilities can be added to the activation link list by selecting an ability in the dropdown next to the “Activate Links” label and then pressing the “+” button. The ability can be removed from the activation link list by hitting the “X” button at the end of the row of each ability. Abilities can be ordered using the up and down arrows at the end of the row of each ability. Once the master ability activates it will then activate the linked abilities in order of the list.

Effects

Overview

In this section effects can be setup which will be applied to other objects which have the ABC StateManager component. Effects are applied from ability hits/collisions. Effects will be inflicted one by one to the entity in the order that they appear in this section, effects can be reordered.

Each effect is uniquely attached to the ability and can only be added to an entity if they do not already have the same effect from the same ability applied on them (unless the effect has the “Allow Duplicate Effect Activation” setting enabled).

For example: an entity has a Fireball ability and a Lightning ability and both abilities have a Adjust Health effect setup. Even though the 2 effects do the same thing they are unique and not related. If the entity activates 2 Fireball abilities and 1 Lightning ability and all abilities hits the enemy at the exact same time then the enemy will be inflicted with the Adjust Health effect from the Lightning and 1 Fireball ability. The effect from the second fireball ability will not be applied as the enemy would already have the Fireball effect active on him.

Each effect has a number of settings and graphics which can play out which provide a number of customisable possibilities.

Details on what each effect does can be found by clicking the following link: Ability Effects.

Enable Ability Effect Links

This setting if enabled will allow for the ability to set up ability effect links. Any abilities linked will have all their effects applied on any ABC StateManager objects hit by the master ability (which is currently being configured). The master ability will also add its own effects if they have been setup.

For example if 2 abilities are created Fireball and Lightning. You can have a Adjust Health effect setup on the lightning ability. If you then configured the Fireball ability and added Lightning as an effect link, then every time the Fireball hits an object it will apply all effects setup for the Lightning ability. The same rules apply regarding not being able to apply the same effect from the same ability on an entity who already has the effect active on them (unless the effect has the “Allow Duplicate Effect Activation” setting enabled). The effect will just act as if the Lightning applied the effect and will not act any different just because it was applied through being linked to the Fireball ability.

This can be useful if you want to set out certain effects which you may want to change once but be applied to a number of abilities. You are more than free to create an ‘effect abilities’ which will never be activated in game but just hosts effects which are linked to by other abilities.

Once enabled the Ability Effect Links list box will appear. Abilities (which will have all their effects linked) can be added to the list by selecting an ability in the dropdown next to the “Effect Links” label and then pressing the “+” button. The ability can be removed from the effect link list by hitting the “X” button at the end of the row of each ability. Abilities can be ordered using the up and down arrows at the end of the row of each ability. The linked ability effects will be applied one by one by in order of the list.

Adding Effects

Effects can be added by using the dropdown present at the top of the section. The drop down lists all predefined effects which are ready to use within the ABC section and ranges from adjusting health and moving objects to enabling abilities and removing other effects. There is also a default “New” option which adds an empty effect name which can be renamed if you have coded in additional effects yourself.

Once a dropdown option has been added pressing the “+” button will add the effect below. There is no limit to the amount of effects that can be added. The effects are listed one after another with their own section of customisable settings. Effects will be applied one by one to the entity in the order that they appear in this section, effects can be reordered.

Below lists the settings which will appear for each effect that is added.

More Settings (Toggle)

A toggle which when pressed will show/hide more settings for the effect. If toggled off then only a few settings for quick editing can be seen.

Potency

The potency of the effect i.e how much to adjust health by. This is the power/force/value behind the effect. Details on how potency is used for each effect can be found in the Ability Effects section.

Start Delay

The delay before the effect will be applied to the entity. Effects are added in order from top to bottom one by one from the effect list. Any delays will prolong the other effects which are applied after.

For example:

  • Effect 1 has delay of 5 so system waits 5 seconds
  • Effect 1 is applied
  • Effect 2 has delay of 2 so system waits another 2 seconds (7 seconds total so far)
  • Effect 2 is applied.
  • Effect 3 has 0 delay so is applied instantly (but after 7 seconds due to other delays)

Duration

How long the effect will be active for before it’s removed from the entity. The same effect from the same ability can not be reapplied if the effect is currently active on the entity (unless the effect has the “Allow Duplicate Effect Activation” setting enabled). Recommend effect duration is kept low if an ability is going to be activated in quick succession and you want the effects to apply rapidly on targets with low restriction (machine gun hits).

If an ability effect like a stat enhancement has been applied then you might want a greater duration so the same stat enhancement effect can’t be added repeatedly until the original effect applied wears off, this then ensures that the player can’t keep activating the ability for really high stats.

If this setting is set as -1 then the effect will never be removed after a duration, it will have an infinite duration and can only be removed through other means (i.e from another effect).

Up / Down

Will move the effect up or down the list changing the order in which the effect is applied.

Red “X”

Will delete the effect removing it from the ability.

ID

An automatically generated number which links to the effect. This is used by the system to always identify the ability effect even if the effect name changes etc.

Effect Name

The name of the effect. This will be used later on by the system to determine which code to run when the effect is applied. This box will be autofilled with the same name as the dropdown option selected but can be modified with any text. If the effect name does not match any code then no effect will occur however the graphics etc will still play out.

Details on what each effect does can be found in the Ability Effects section.

It’s possible to add extra effect code to the ABC system manually or through event delegate subscriptions from other scripts. The effect name is used to differentiate which code to run.

Enable Remove Effect

If enabled then once the effect is removed from the entity it will run the remove code for the effect name set. Most the time this code will return the entity back to the state it was at before the effect was applied. I.e if a stat enhancement effect was applied adding 30 attack then once the effect is removed, if this box is ticked it will reduce 30 from the attack.

Please refer to the Ability Effects section for details on what happens for each effect when "remove effect" is enabled.

Dispellable

If ticked then the effect can be dispelled (removed earlier) through other effects.

Activate On

Determines who the effects actions will be processed on. Although the effect is applied, activated and tracked against the entity hit. The actual effect actions/code can be setup to either run against the originator who activated the ability in the first place, the entity which had the effect applied or both simultaneously.

This can be used to set up a drain type ability which for example has 2 effects “Adjust Health” with a -potency which will take health and damage from the entity hit and then another “Adjust Health” effect with a +potency which will recover health on the originator who activated the ability. Simulating that the ability drains the health from the entity hit which then gets given to the entity that activated the ability restoring health.

A dropdown with 3 options: “Entity”, “Originator” and “Both Simultaneously“ can be selected to choose who the effect actions will run on.

Misc Property

An additional box which can take any type of value as a second ‘parameter’ of data which is required for some effects. I.e For the “AdjustGroupEnablePoint” effect, the potency is how many points to add and the Misc Property box expects the ID of the Ability Group to add the points too.

Please refer to the Ability Effects section for details on what the Misc Property does for each effect. Not all effects require an Misc Property.

Alt Property

Another additional box which can take any type of value as a third ‘parameter’ of data which is required for some effects. I.e For the “Push” effect, the potency is how far back the entity will be pushed, the Misc property explained above determines how long it takes for the entity to reach the push distance and the Alt property sets how much lift is produced by the push.

Please refer to the Ability Effects section for details on what the Alt Property does for each effect. Not all effects require an Alt Property

Event Type

Determines the event type when activating and deactivating the effect. “Standard” will run the normal code integrated with the ABC system. “RaiseEvent” will only invoke the event delegate and “Both” will raise the event and run the normal code.

Note: The deactivating functionality will only run if the “Enable Remove Effect” setting is enabled.

If “Both” or “RaiseEvent” is selected then an event will be raised informing all scripts which have subscribed that the effect has been applied or removed.

The event will pass through the Effect object (with all its settings defined in this section) the Target entity where the effects have been applied and the Originator who activated the ability.

Code example to subscribe to the event delegate: 

                    
                          private void OnEnable() {
                            //Record transform
                                meTransform = transform;

                                //Get Statemanger script
                                if (playerStateManager == null)
                                    playerStateManager = meTransform.GetComponent<ABC_StateManager>();

                                //Subscribe to event
                                playerStateManager.onEffectActivation += EffectListener;   
                            }

                            private void EffectListener(Effect effect, ABC_IEntity Target, ABC_IEntity Originator) {
                                switch (effect.effectName) {
                                    case "CustomEffect":
                                        //add custom effect here
                                        break;
                                }
                            }

                            private void OnDisable() {
                                //unsubscribe from event
                                playerStateManager.onEffectActivation -= EffectListener;
                            }

A component has been created called ABC_EffectEventSubscriber which can be attached to any entity which will aid in linking ability effects to other script functionality. This component also highlights within the code comments how to add custom effects with additional code. More information the on the ABC_EffectEventSubscriber component can be found here: Effect Event Subscriber.

Effect Random Chance

If ticked then there is a random chance that the effect will not be applied, dependant on a probability min and max scale set. Once enabled the probability min and a probability max bar will appear.

Before the system applies the effect to the entity it will roll a dice for a value, if the value sits within the min and max probability then the effect will be successfully applied, else it will not be applied at all. The higher the Min/Max gap the higher the chance it will be applied on the entity.

Hit Stops Movement

If enabled then the entities movements will be stopped for a duration when the effect is successfully applied, this requires the “Hits Can Stop Movement” global setting to be enabled in the entities ABC StateManager component. Once ticked the following settings will appear:

  • Stop Movement Chance
      If ticked then there is a random chance that the movement restriction will not occur, dependant on a probability min and max scale set. Once enabled the probability min and a probability max bar will appear.
      Before the system applies the effect to the entity it will roll a dice for a value, if the value sits within the min and max probability then the movement restriction will be successfully applied for a duration, else it will not be applied at all. The higher the Min/Max gap the higher the chance the entities movement will be stopped.

Hit Prevent Activation

If enabled then the entities ability activation will be restricted for a duration when the effect is successfully applied, this requires the “Hits Can Prevent Activation” global setting to be enabled in the entities ABC Controller Settings. Once ticked the following settings will appear:

  • Prevent Activation Chance
      If ticked then there is a random chance that the ability activation restriction will not occur, dependant on a probability min and max scale set. Once enabled the probability min and a probability max bar will appear.
      When the effect is applied the system will roll a dice for a value, if the value sits within the min and max probability then the activation restriction will be successfully applied for a duration, else it will not be applied at all. The higher the Min/Max gap the higher the chance the entity's ability activation will be stopped.

Hit Interrupt Activation

If enabled then any abilities the entity is currently activating can be interrupted when the effect is successfully applied, this requires the “Hits Can Interrupt Activation” global setting to be enabled in the entities ABC Controller Settings. Once ticked the following settings will appear:

  • Interrupt Activation Chance
      If ticked then there is a random chance that the ability activation interruption will not occur, dependant on a probability min and max scale set. Once enabled the probability min and a probability max bar will appear.
      When the effect is applied the system will roll a dice for a value, if the value sits within the min and max probability then any abilities the entity is currently activating will be interrupted successfully, else no abilities will be interrupted. The higher the Min/Max gap the higher the chance the entity's ability activation will be interrupted.

Modify Potency Using Stats

If enabled then the system will use stats from either the target entity and/or the originator who activated the ability to modify the potency value. Stats can be created in the ABC StateManager configuration, for information on setting up stats please click the following link: Entity Stats

This settings allows for entities stats to affect the power/force/value etc behind an effect. For example with an “AdjustHealth” effect the originators attack stat may modify the potency reducing more health (doing more damage) however the effect may also have it’s potency modified again depending on the targets defence so it doesn’t reduce as much as it would have.

Details on what potency does for each effect can be found by clicking the following link: Ability Effects.

There is no limit to how many Stat Modifications that can be added to an effect. Modifications can be added by pressing the “+” button. The modifications can be removed from the list by hitting the “X” button at the end of the row. Modifications can be ordered using the up and down arrows at the end of the row of each ability. The stat modifications will be applied one by one in order of the list.

Once a modification has been added the following settings appear in a line:

  • Add/Minus/Divide/Multiply the stat
      The athromitic being applied to the potency (increase/decrease potency etc)
  • Percentage Of
      What percentage of the stat will be used in the calculation. I.e Add 30% of the Attack stat for example If Attack stat is 100 it will add 30 to the potency.
  • Originator (entity that activates the ability) / Entity (entity hit by the ability) / Both Simultaneously
      Who’s stat is being used in the calculation. A choice of Originator, Entity or Both can be used. For example use 30% of the Originators Attack stat.
  • Stat
      The name of the stat to use. The string needs to match the stat which has been setup in the ABC StateManager configuration or it will be ignored.

An example of these settings being used can be found below. 2 Stat modifications are shown which take into account the originators attack stat and the target entities defence stat to work out how much health is reduced from the target entity.

Starting Potency -100 (reduce health by 100 as a base)

Modification 1: Minus 30% of Originators Attack (100) (-100) - (30) = -130

Modification 2: Add 100% of Entity Defence (20)

(-100) - (30) + (20) = -110

The potency is decreased by 30% of the originator attack (negative value/decreased as effect is adjusting health) but then is then increased by 100% of the entity defence making it -110 adjusted health i.e 110 damage the target will receive.

Allow Duplicate Effect Activation

If enabled then this effect can be applied multiple times on an entity no matter if they already have the effect active. If unticked then this abilities effect can only be applied to entities who don’t currently have the effect applied to them.

If ticked the following settings will appear:

  • Limit Duplicate Effect Activation
      Will limit the amount of duplicate effects that can be active on the entity at once
  • Max No Of Duplicate Effects
      The maximum number of duplicate effects that can be active on the entity at once. Once the max has been reached any further effects will not be applied until the number of duplicate effects is back below the max.

Activation Range

If true then the effect will only be applied and activate on the target entity if they are still in a set range from the ability projectile current position/last known position before it was destroyed.

Works well for adding dodging mechanics to games. I.e A fireblast ability may do initial damage on contact but then leave behind a flame which will do more damage from an effect setup to activate after a 5 second delay, using this setting the system will only apply the second damage effect if the target entity is still standing in the flames i.e a set range from the fireblast. If they are outside the range they won’t get hurt from the second effect.

If enabled the following settings appear:

  • Use Projectiles Range
      Instead of setting the “Effect Activation Range” setting this will instead take the range depending on the projectile graphic size
  • Effect Activation Range
      A float value can be set which will determine the range the entity needs to be within from the projectile current position/last known position for the effect to be applied and activated.

Repeat Effect

If enabled then the effects action will be repeated on the entity for a set duration every defined interval. The repeat effect action will stop early if the effect is removed from the entity even if it’s before the amount of times the effect is set to repeat.

For example it can be setup so an AdjustHealth effect will repeat every 2 seconds for the next 10 seconds, which will result in the entity taking damage every 2 seconds for a 10 second period during play this can simulate a burn effect from a fireball ability or a poison infliction from another ability.

If enabled the following settings appear:

  • Repeat Aesthetics
      If true then the effect graphics will activate every time the effect action is repeated
  • Repeat Duration
      The max duration in which the effect action can be repeated, i.e only allow repeat actions within the next 10 seconds.
  • Repeat Interval
      The interval between effect actions, i.e how often the effect actions will repeat within the repeat duration.

Effect Graphic

If enabled then a graphic will play when the effect is applied and activated. Once ticked the following settings appear:

  • Main Graphic
      Any object can be added here which will appear when the effect activates. A particle object for example can be used to simulate an explosion etc. If no object is added then no graphic will appear.
  • Sub Graphic
      A second object can be added here which will become a child of the Main Graphic described above. This object like the main graphic will appear whenever the effect is activated.
  • Graphic Duration
      How long the graphic will last for. Unlike other setting If 0 is entered then the graphic will not be infinite and will disappear straight away after 0 seconds.
  • Graphic Delay
      How long after the effect is activated that the graphic will show.
  • Follow Affected Object
      If enabled then the graphic will become a child of the entity that the effect was activated on. This will naturally mean the graphic will move with the entity.
  • Effect On Hit Position
      If true then the effect graphic will appear where the ability collision hit occured (if the data is available). If false or the hit point data is not provided then it will be at the targets transform position.
      Abilities using trigger collision are not able to collect contact data so will default to the targets transform position. More settings on turning trigger collision on and off can be found by clicking the following link: Is Trigger
  • Graphic Offset
      X, Y, Z float values can be entered which will modify where the graphic object is positioned from it’s starting position. I.e 90 in the X box will move the object up the X axis by 90 from the position it starts at.
  • Forward Offset
      A float value can be entered here to move the graphic object forward from it’s starting position. If negative value is entered it will move the graphic backwards rather then forwards.
  • Right Offset
      A float value can be entered here to move the graphic object to the right from its starting position. If negative value is entered it will move the indicator left rather then right.

Effect Ignore Tags

ABC or Unity tags can be defined which will tell the system not to apply the effect to any entity that has a matching tag. 2 tick boxes can be ticked detailing what part of the entity is checked for tags:

  • Check Tags On Entity
      If enabled then the system will try to find a ABC/Unity Tag on the entity target hit by the ability which has the ABC StateManager Component, if a tag matches any Effect Ignore Tags set then the effect will not be applied on the entity
  • Check Tags On Object Hit
      If enabled then the system will try to find a ABC/Unity Tag on the specific gameobject which was hit by the ability which doesn’t necessarily have the ABC StateManager component but is a child object of the entity which does have the component. if a tag matches any Effect Ignore Tags set then the effect will not be applied on the entity

An example of a use for this functionality can be for FPS games and headshots. You can tag each head child object on an entity as “Headshot” then stop the normal effect from applying damage if the object hit has a “Headshot” tag as a later effect is going to do more damage which has been setup to activate on objects hit with a “Headshot” tag

Overview

  • Max Active
      The limit to how many versions of this ability can be active at one time.

                                        }

                                        public void EnableNavigation() {
                                        //Code to enable Navigation
                                        }


                                        public void DisableNavigation() {
                                        //Code to disable DisableNavigation
                                        }


                                private void OnDisable() {
                                    if (this.movementEventsABC != null) {
                                    this.movementEventsABC.onEnableMovement -= this.EnableMovement;
                                    this.movementEventsABC.onDisableMovement -= this.DisableMovement;
                                    }

Raise Ability Activation Completed Event

If ticked then a ability activation completed delegate event will be raised when the ability has either finished initiating or has been destroyed. This event can be connected to from any script. This way any additional scripts outside of the system can know when an ability has finished activating. Once ticked the following setting will appear:

  • Ability Activation Completed Event Type
      This allows you to choose when the ability activation completed event is raised. There is 2 choices: "Ability Destroyed" which raises the event after the ability is disabled through collision or duration or "Ability Initiated" which is raised as soon as the entity has initiated the ability (the spell has left his hands)

    If the ability is cancelled/interrupted then the completed event will always raise no matter what event type is chosen

Code example to subscribe to the event delegate: 

                    
                               private void OnEnable() {
                                     this.ABCEventsCont = transform.GetComponentInChildren();

                                        if (this.ABCEventsCont != null)
                                        {
                                            this.ABCEventsCont.onAbilityActivation += this.DisableNavigation;
                                            this.ABCEventsCont.onAbilityActivationComplete += this.EnableNavigation;
                                        }

                                        public void EnableNavigation() {
                                        //Code to enable Navigation
                                        }


                                        public void DisableNavigation() {
                                        //Code to disable DisableNavigation
                                        }


                                private void OnDisable() {
                                    if (this.movementEventsABC != null) {
                                    this.movementEventsABC.onEnableMovement -= this.EnableMovement;
                                    this.movementEventsABC.onDisableMovement -= this.DisableMovement;
                                    }
                                }

Include Surrounding Objects

If enabled then nearby objects can be converted to a Surrounding Object. Surrounding Objects will become attached to the ability, travelling along side it. Once the ability is destroyed all surrounding objects will be converted back to the states they were in before unlinking them from the ability (unless they are destroyed from ability effects or the “Destroy On Collision” Setting detailed below).

For example any debris in the game can be setup to travel with an ability projectile, simulating a force moving the debris like telekinesis. Only objects with the ABC StateManager component with the “Block Surrounding Object Status” unticked can be converted to a surrounding object.

Surrounding Objects will not work for Raycast ability types as no ability object is produced for the surrounding objects to travel with.

There is no limit to how many surrounding objects can be used. You can convert nearby environments or enemies into a surrounding object which will gravitate to and travel with the ability. There is also options to apply the abilities effects or destroy surrounding objects when the ability collides.

When the checkbox is enabled the below settings will appear.

Lock To Projectile

If enabled then any surrounding objects created and linked to this ability can not be ‘stolen’ by another ability using surrounding objects.

Minimum Scatter Range

Surrounding objects will scatter/hover around the ability so they are never directly on top or in the way. The scatter range is determined by the min and max value provided.

Maximum Scatter Range

Surrounding objects will scatter/hover around the ability so they are never directly on top or in the way. The scatter range is determined by the min and max value provided.

Send Object To Projectile

If enabled then any surrounding objects created will first travel towards the ability over a duration, once the object reaches the projectile it will then travel along side it. If disabled then the surrounding objects will instead instantly appear at the ability ready to travel along side it.

If enabled but no duration is provided then the object will just hover at the position it was at before being converted to a surrounding object, still travelling along side the ability in the same direction etc

If enabled the following setting will appear:

  • Object To Projectile Duration
      Defines how long it will take the for the travelling surrounding object to reach the ability.

Enable Collider

If ticked then all the surrounding objects will be able to collide and detect collisions as if they were part of the ability. I.e if a surrounding object hits an entity then ability effects will be applied and collision events will occur as if it was the ability that hits the entity and not the object.

If enabled the following settings will appear:

  • Enable When Projectile Reached
      If enabled then the surrounding objects will start detecting collision only once it reaches the ability (i.e when “Send Object To Projectile” is enabled). All collisions that occur before the surrounding objects reach the ability will be ignored.

Effect On Collision

If enabled then when the ability collides it will apply and activate all effects to its surrounding objects. For example if enemy objects have been converted to a surrounding object and then the ability collides it will apply all the effects on the enemy objects which might include health adjustments etc which simulates damage due to the enemy being smacked into a wall etc.

Destroy On Collision

If enabled then when the ability collides it will destroy and disable its surrounding objects. For example if debris objects have been converted to a surrounding object and then the ability collides it will destroy all the debris etc which simulates the debris breaking due to the being smacked into a wall etc.

Surrounding Tags

If enabled then ABC StateManager objects near the ability starting position can be converted to a Surrounding Object if they have an ABC/Unity Tag which matches any tags added to the Affected Tags list. Only a certain number of surrounding objects can be converted determined by the “Object Limit” setting.

Note: Only objects with the ABC StateManager component with the “Block Surrounding Object Status” unticked can be converted to a surrounding object.

If ticked the following settings will appear:

  • Surrounding Range
      The range in which objects are checked around the ability starting position. Only objects within this range have the potential to become a Surrounding Object
  • Surrounding Object Required
      If enabled then the ability will only activate if atleast 1 surrounding object has been created from matching tags.
      If no objects are converted as no tags match then the ability activation will be interrupted
  • Object Limit
      The max amount of objects that can be converted to a surrounding object through matching tags.
  • Affected Tags
      Clicking the “+” sign will create a new text box to allow for Tags to be entered. You can then delete a tag by clicking the “x” or change the ordering with the arrow keys. Both ABC and Unity tagging systems are used when trying to match Tags.

Surrounding Target Select

If enabled then the object that the entity is currently targeting can be converted to a Surrounding Object.

Note: Only objects with the ABC StateManager component with the “Block Surrounding Object Status” unticked can be converted to a surrounding object.

If ticked the following settings will appear:

  • Surrounding Range
      The range in which the entities target has to be within to become a Surrounding Object.
  • Auxiliary SoftTarget
      If ticked then in the scenario that the entity does not currently have a selected target then the entities current soft target will become a Surrounding Object instead.
  • Surrounding Object Required
      If enabled then the ability will only activate if a surrounding object has been created from the entities current target/soft target.
      If no objects are converted as the entity had no target/soft target then the ability activation will be interrupted
  • Only Affect Tag
      If enabled then the entities current target can only be converted to a Surrounding Object if they either have have an ABC or a Unity tag which matches any tags defined in the list box.
      Clicking the “+” sign will create a new text box to allow for Tags to be entered. You can then delete a tag by clicking the “x” or change the ordering with the arrow keys. Both ABC and Unity tagging systems are used when trying to match Tags.
